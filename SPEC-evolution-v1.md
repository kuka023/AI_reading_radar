# SPEC · 雷达自进化路线 v1

> 状态：v0 提案，供业务与开发评审。飞轮阈值为初始建议值，待专项讨论后修订。

## 0. 定位

雷达的「AI 能力」不体现为再造一个通用书单，而体现为**在人的把关下让书库与推荐持续自我进化**。

**核心原则（贯穿全部三期，也是产品护城河）：**

> **AI 负责聚合信号、发现候选、匹配排序、起草内容；人负责准入书目、定稿路径、维护分类法。**

一旦允许 AI 自动把书放入雷达、或自动生成学习书序，产品即退化为一个通用 LLM 书单，「有观点的策展 + 防幻觉」的定位随之失效。因此每一处 AI 产出，后面都必须有一个人工闸门。

## 1. 本轮已确认的决策

| 决策项 | 结论 | 说明 |
|---|---|---|
| 成长路径作者 | 初期人工撰写；积累数据后由 AI 辅助生成 | AI 仅从信号中推荐候选书，人工排序与定稿 |
| 个性化目标输入 | 初期采用结构化选择（对齐能力主题）；后续再叠加自由文本 | |
| 飞轮准入/降级阈值 | 采用本文 §3 的 v0 提案，后续讨论后修订 | |
| 实施顺序 | 成长路径 → 飞轮规则 → 个性化推荐 | 依据冷启动约束，见 §2 |

## 2. 三段闭环与分期

三个方向构成同一个 AI 循环，且对数据的依赖程度不同，决定了实施顺序。

| 阶段 | 方向 | 作用面 | 数据依赖 | 建议顺序 |
|---|---|---|---|---|
| 供给侧 | 飞轮进化书库 | 决定雷达里有哪些书 | 高 | 二 |
| 连接层 | 成长路径 + 讨论 | 产生信号，且即时可用 | 低 | **一** |
| 需求侧 | 差异化个性推荐 | 决定给谁推哪本 | 最高 | 三 |

**冷启动判断：** 成长路径由人工策展，零用户数据即可交付价值，且顺带产出个性化所需的「目标」输入；飞轮与个性化都要等信号密度足够才成立。故先做成长路径。

## 3. 飞轮阈值 v0（供给侧，本轮重点）

### 3.1 可用信号

| 记号 | 含义 | 来源 |
|---|---|---|
| V | 详情页浏览人数 | events（open） |
| R | 标记「读过」人数 | shelf.state=read |
| S / s̄ | 带分推荐条数 / 推荐分均值（1–5） | reviews.score |
| useful_rate | 有用度 = (很有用 + 0.5·一般) / 有反馈总数 | shelf.useful |
| conversion | 读过转化 = R / max(V,1) | 派生 |

### 3.2 准入（候选书 → 雷达）

候选来源：用户从图书馆加入个人图层的外部书、社区推荐中尚未进雷达的书。

**同时满足以下四项，进入「专家评审队列」：**

- R ≥ 8（有足够多的人真正读过，而非仅收藏）
- S ≥ 5（有足够多的带分推荐）
- s̄ ≥ 4.2（推荐分均值）
- useful_rate ≥ 0.70（对目标有用的占比）

**再由专家评审**：判断契合度、质量、是否与在册书重复，并指定能力族 / 主题 / 立场 / 难度。仅在评审通过后写入雷达（curated=1）。人工闸门不可省略。

### 3.3 降级 / Hold（雷达书 → 降级或进 Hold）

飞轮必须对称——只进不出会使雷达持续膨胀。

**满足任一项，进入专家评审队列（降级或 Hold 决策）：**

- R ≥ 8 且 s̄ ≤ 2.5（读过的人普遍评价低）
- useful_rate ≤ 0.35（有效反馈 ≥ 8 人时）
- 衰变引擎判定为过时（有新版取代 / 绑死废弃框架 / 技术路线被取代）——沿用现有纯规则引擎

### 3.4 防「热门更热门」偏差

裸星级会让样本量大的热门书天然占优，冷门好书难以浮现。因此：

- 准入队列**按 useful_rate 排序**，而非按浏览量或推荐条数排序，使「读过的人转化率高」的书优先于「看的人多但评价平庸」的书。
- 评审关注 conversion 与 useful_rate，而非绝对热度。

### 3.5 节奏与冷启动

- **评审节奏**：每两周处理一次队列，或队列累积到一定条数时触发。初期人工、低频。
- **冷启动**：在活跃评分用户达到约 50 人之前，上述阈值实际上无法触及，准入以**专家手工加书**为主，自动标记逻辑处于休眠。阈值数值是**成熟期的占位值**，需随真实流量校准。

> §3 全部数值为 v0 起点，待专项讨论后修订。

## 4. 成长路径设计（Phase 1 · 连接层）

### 4.1 数据模型

```
path = {
  id, title,            例：AI 产品入门
  role,                 能力族（对齐 roles）
  level,                入门 / 进阶 / 高阶
  goal,                 一句话目标
  steps: [ { resource_id, why } ],   有序，每步附「为什么这一步」
  author,               撰写人（初期为人工）
  status                draft / published
}
```

### 4.2 初期路径清单（骨架，书序待专家定稿）

以下仅为容器示例，**具体书序需由你 / 领域专家撰写与审定**，AI 不生成书序。

- **AI 产品入门**（产品 · 入门）
- **Agent 开发入门**（技术·应用 · 入门）
- **AI 产品从入门到高阶**（产品 · 入门→高阶，在「入门」基础上叠加进阶书）

### 4.3 人工与 AI 分工

- **初期**：人工撰写路径标题、目标、书序与每步理由。
- **后期（有数据后）**：AI 从飞轮信号中为某目标**推荐候选书与顺序**，人工排序定稿。AI 辅助，人工闸门不变。

## 5. 差异化个性推荐设计（Phase 3 · 需求侧）

### 5.1 输入（初期结构化）

- 能力族（角色）
- 目标：从能力主题派生的**结构化选项**（例：「上线一个 RAG 应用」「入门 Agent 系统」）
- 时限（例：30 天）
- 现有水平 / 阅读史（来自书架与事件）

### 5.2 机制（防幻觉铁律）

- 个性化**只在策展书库内重排序与规划路径，绝不生成书名**。
- 依据「角色 + 目标主题 + 阅读史」筛选并排序：立场（现在就读优先）→ 难度升序 → 结合页数/预计时长，规划出在时限内可完成的书序。
- 输出必须携带与雷达一致的「**为什么**」（立场 + 理由），保持可解释。黑箱推荐会削弱「有观点」的定位。
